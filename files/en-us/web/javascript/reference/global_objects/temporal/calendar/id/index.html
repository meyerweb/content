The id read-only property returns an unambiguous identifier for the calendar.When subclassing {{jsxref('Temporal/Calendar','Temporal.Calendar')}}, this property doesn't need to be overridden because the default implementation gives the result of calling {{jsxref('Temporal/Calendar/toString','Temporal.Calendar.toString()')}}.

Syntax

calendar.id;

Value

A string containing an unambiguous identifier for the calendar. Effectively, this is the canonicalized version of whatever calendarIdentifier was passed as a parameter to {{jsxref('Temporal/calendar/constructor','the constructor','','nocode')}}.

Examples

date = Temporal.Calendar.from('gregory');
date.id; // => "gregory"
